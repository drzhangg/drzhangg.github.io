networknamespace 通过 bridge 连接公网

熟悉一下在linux中，通过命令行创建network namespace虚拟网络，访问到外网

# 1.创建network namespace

ip netns add net1

# 2.创建veth对

# 创建 veth0和veth1 一对veth
ip link add veth0 type veth peer name veth1

# 3.配置net1

# 把 veth0插入到上面创建的ns  net1中
ip link set veth0 netns net1

# 给net1中的veth0配置上ip
ip netns exec net1 ip addr add 192.168.0.2/24 dev veth0

# 启动net1中的veth0
ip netns exec net1 ip link set veth0 up

# 4.创建bridge

ip link add br0 type bridge

# 5.配置bridge

# 给br0配置上ip
ip addr add 192.168.0.1/24 dev br0

# 把veth1接入到bridge中
ip link set dev veth1 master br0   # master 是一个关键字，用于将一个网络接口设置为另一个网络设备（通常是网桥）的从属成员

# 启动 veth1
ip link set veth1 up

# 启动 br0
ip link set br0 up

# 6.尝试请求外部资源

$ ip netns exec net1 ping 10.153.*.*
connect: Network is unreachable  #网络不可达


# 查看下路由
ip netns exec net1 route -n
Kernel IP routing table
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
192.168.0.0     0.0.0.0         255.255.255.0   U     0      0        0 veth1

这里因为net1中还没有配置路由，默认只有192.168.0.* 这个网段的路由规则，我们ping的ip是 10.153..，根据路由表找不到出口，所以导致的失败。

# 7.配置路由


ip netns exec net1 route add default gw 192.168.0.1 veth0

这里配置了一条默认路由，只要匹配不到其他规则，就默认转发到veth0上，同时指定下一条是它所连接的Bridge（192.168.0.1）

# 再ping一下
ip netns exec net1 ping 10.153.*.* -c 2
PING 10.153.*.* (10.153.*.*) 56(84) bytes of data.

--- 10.153.*.* ping statistics ---
2 packets transmitted, 0 received, 100% packet loss, time 999ms

仍然不通，上面的路由帮我们把数据包从 veth 正确送到了bridge网桥。接下来，网桥还需要Bridge转发到eth0上。所以还需要打开两个转发相关的配置。

# 8.配置转发

sysctl net.ipv4.conf.all.forwarding=1

iptables -P FORWARD ACCEPT

不过这个时候，还存在一个问题。那就是外部的机器并不认识192.168.0.* 这个网段的ip。他们都是通过 10.* 进行通信的。这时候需要使用SNAT进行配置。

# 9.配置SNAT

这次的需求是实现内部虛拟网络访问外网，所以需要使用的是SNAT。它将namespace请求中的ip(192.168.0.2) 换成外部网络认识的10.153..，进而达到正常访问外部网络的效果。

iptables -t nat -A POSTROUTING -s 192.168.0.0/24 ! -o br0 -j MASQUERADE

再ping一下试试

PING 10.153.*.* (10.153.*.*) 56(84) bytes of data.
64 bytes from 10.153.*.*: icmp_seq=1 ttl=250 time=3.77 ms
64 bytes from 10.153.*.*: icmp_seq=2 ttl=250 time=3.55 ms

可以看到，已经能正常通信了。

# 开放端口

我们再考虑另外一个需求，那就是把在这个网络命名空间内的服务提供给外部网络使用。和上面的问题一样，虚拟网络环境中 192.168.0.2这个ip外界是不认识它的。只有这个宿主机知道它是谁，所以我们同样还需要NAT功能。

这次我们是要实现外部网络访问内部地址，所以需要的是DNAT配置。DNAT和SNAT配置中有一个不一样的地方就是需要明确指定容器中的端又在宿主机上对应哪个。

通过下面这个命令来配置DNAT规则。


iptables -t nat -A PREROUTING ! -i br0 -p tcp -m tcp --dport 8088 -j DNAT --to-destination 192.168.0.2:80

以上就是通过创建network namespace，配置bridge来和外部通信的过程了。
