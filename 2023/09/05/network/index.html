网络相关

二层网络

二层网络也称数据链路层网络，主要工作在 OSI 的第二层，即数据链路层。

主要用于连接不同物理网络之间的通信。二层网络通常基于MAC地址来进行数据帧的转发，它将数据帧从一个物理网络转发到另一个物理网络，以实现跨网络的通信。

特点：

  • 基于 MAC 地址进行通信，适用于局域网内部设备的通信。
  • 使用交换机进行数据帧转发，通过学习 MAC 地址和端口的映射来实现。
  • 不涉及 IP 或路由表，因此在设备通信方面更快速，但通常局限于同一局域网内。

三层网络

三层网络也称为网络层网络，工作在 OSI 的第三层，即网络层。

主要用于在不同的 IP 子网之间进行路由和通信。三层网络使用IP地址来确定数据包的路由，它能够实现更复杂的网络拓扑和子网之间的互连。

特点：

  • 基于IP地址进行通信，适用于不同子网和网络之间的通信。
  • 使用路由器进行数据包路由，根据路由表决定数据包的转发路径。
  • 支持更大范围的网络通信，能够实现不同子网之间的连接和互通。

overlay

  • 特点：overlay是基于underlay实现的虚拟网络。它使用隧道技术将数据包封装在underlay网络中，从而创造了一个逻辑上的虚拟网络，提供了额外的网络抽象、隔离、安全性等功能。
  • 区别：overlay 网络允许在不同主机上的容器之间建立虚拟的、隔离的网络通信，同时隐藏了底层网络的复杂性。它提供了更大的灵活性和可移植性，允许在不同的物理网络环境中使用相同的网络配置。

underlay

  • 特点：underlay是指底层物理网络，即实际的网络基础设施。它是由网络设备（如交换机、路由器）和物理链路组成的网络基础架构。
  • 区别：underlay网络负责在物理主机之间提供基本的网络互连功能。它是overlay网络的基础，为overlay网络提供了传输容器之间数据流的基本路径。

容器网络中的underlay网络是指借助驱动程序将宿主机的底层网络接口直接暴露给容器使用的一种网络构建技术，较为常见的解决方案有macvaln、ipvlan和直接路由等。

VXLAN

VXLAN（Virtual eXtensible Local Area Network，虚拟可扩展局域网），是一种虚拟化隧道通信技术。它是一种 Overlay（覆盖网络）技术，通过三层的网络来搭建虚拟的二层网络。

简单来说，VXLAN 是在底层物理网络（underlay）之上使用隧道技术，借助 UDP 层构建的 Overlay 的逻辑网络，使逻辑网络与物理网络解耦，实现灵活的组网需求。

VXLAN 不仅支持一对一，也支持一对多，一个 VXLAN 设备能通过像网桥一样的学习方式学习到其他对端的 IP 地址，还可以直接配置静态转发表。

VXLAN 的几个重要概念：

  • VNI（Virtual Network Identifier）：VXLAN VNI是一个24位标识符，用于区分不同的VXLAN网络。每个虚拟网络会被分配一个唯一的VXLNA VNI。VXLNA VNI的引入允许在共享的物理网络上创建多个逻辑上隔离的VXLAN网络。
  • VTEP（VXLAN Tunnel Endpoints）：VTEP是VXLAN隧道的端点，可以是物理设备或虚拟设备。它负责封装和解封装VXLAN数据包，并将数据包在物理网络中传输。每个VTEP会维护一个MAC表，用于映射虚拟网络中的MAC地址到VXLAN VNI，以确定数据包的目标。
  • Tunnel（VXLAN 隧道）：隧道是一个逻辑上的概念，在 VXLAN 模型中并没有具体的物理实体向对应。隧道可以看做是一种虚拟通道。VXLAN 通信双方认为自己是在直接通信，并不知道底层网络的存在。从整体来说，每个VXLAN 网络像是为通信的虚拟机搭建了一个单独通信通信，也就是隧道。
